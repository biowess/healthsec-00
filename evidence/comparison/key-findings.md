# Key Findings

## 1. Score vs Reality Gap
- Lynis score remained unchanged despite real hardening actions
- Shows limitations of aggregate scoring systems

## 2. Measurable Compliance Improvement
- OpenSCAP detected small improvements (+2 passed rules)
- Confirms targeted configuration impact

## 3. Verified System Changes
- AIDE confirmed multiple filesystem modifications
- Provides strong evidence of applied hardening

## 4. Service-Level Security Gains
- SSH, nftables, and auditd were activated and configured
- Improved system control and observability

## 5. Controlled Network Exposure
- No unintended open ports introduced
- Local service remained properly restricted to loopback

---

## Final Insight

This phase demonstrates that:

- Security improvements are **incremental**
- Different tools measure **different aspects of security**
- Evidence-based validation is more important than score changes
